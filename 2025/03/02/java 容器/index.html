java 容器

18. Java 容器都有哪些？

Java 容器主要分为两大类：Collection 和 Map。

  • Collection（存储单元素）：

    • List（有序、可重复）：

      • ​ArrayList​（动态数组）
      • ​LinkedList​（双向链表）
      • ​Vector​（线程安全的动态数组，已过时）
      • ​Stack​（栈，继承自 Vector​）

    • Set（无序、不可重复）：

      • ​HashSet​（基于哈希表）
      • ​LinkedHashSet​（维护插入顺序）
      • ​TreeSet​（基于红黑树，自然排序）

    • Queue（队列）：

      • ​LinkedList​（也可作为队列）
      • ​PriorityQueue​（优先级队列）
      • ​ArrayDeque​（双端队列）

  • Map（存储键值对）：

    • ​HashMap​（非线程安全，允许 null 键/值）
    • ​LinkedHashMap​（维护插入顺序）
    • ​TreeMap​（基于红黑树，键排序）
    • ​Hashtable​（线程安全，不允许 null）
    • ​ConcurrentHashMap​（线程安全的高并发 Map）

19. Collection 和 Collections 有什么区别？

  • Collection 是集合接口，定义集合的基本操作（如 add()​, remove()​），是 List​、Set​、Queue​ 的父接口。
  • Collections 是工具类，提供静态方法操作集合，如排序（sort()​）、同步包装（synchronizedList()​）等。

20. List、Set、Map 之间的区别是什么？

特性 List Set Map
有序性 按插入顺序排序 无序（HashSet​）或按规则排序（TreeSet​） 键无序（HashMap​）或按规则排序（TreeMap​）
重复元素 允许 不允许 键唯一，值可重复
实现类 ​ArrayList​, LinkedList​ ​HashSet​, TreeSet​ ​HashMap​, TreeMap​
底层结构 动态数组/链表 哈希表/红黑树 哈希表/红黑树
典型用途 需要顺序访问或频繁按索引操作 去重 键值对存储

应用场景：

  • ​List​：用户历史记录（需顺序）。
  • ​Set​：唯一性校验（如用户 ID）。
  • ​Map​：缓存（键快速查找值）。

21. HashMap 和 Hashtable 有什么区别？

特性 HashMap Hashtable
线程安全 非线程安全 线程安全（方法用 synchronized​ 修饰）
Null 支持 允许一个 null 键/多个 null 值 不允许 null 键或值
性能 高（无锁竞争） 低（同步开销）
继承关系 继承 AbstractMap​ 继承 Dictionary​（已过时）
扩容机制 容量翻倍 旧容量×2 +1
哈希算法 位运算优化 直接取模

使用建议：

  • 单线程用 HashMap​，多线程用 ConcurrentHashMap​（而非 Hashtable​）。

22. 如何决定使用 HashMap 还是 TreeMap？

  • HashMap：

    • 适用场景：快速查找（O(1) 时间复杂度）、无需排序。
    • 示例：缓存用户会话信息（键为用户 ID）。

  • TreeMap：

    • 适用场景：需要按键自然顺序或自定义顺序遍历（如按字典序排序）。
    • 示例：统计成绩并按分数排序。

底层结构：

  • ​HashMap​：数组 + 链表/红黑树（链表长度 >8 时转为红黑树）。
  • ​TreeMap​：红黑树（保证有序性）。

23. 说一下 HashMap 的实现原理？

  1. 数据结构：数组 + 链表/红黑树。

  2. 哈希计算：

    • 通过 key.hashCode()​ 计算哈希值。
    • 使用 (n-1) & hash​ 确定数组下标（n 是数组长度）。

  3. 冲突解决：

    • 链表法（同一桶内链表存储）。
    • 链表长度 >8 时转为红黑树（提高查询效率）。

  4. 扩容机制：

    • 默认容量 16，负载因子 0.75，扩容后容量翻倍。
    • 重新计算所有元素的哈希位置（耗时操作）。

PUT 流程：

  1. 计算键的哈希值。
  2. 定位到数组下标。
  3. 若该位置为空，直接插入；否则处理冲突（链表或红黑树）。

24. 说一下 HashSet 的实现原理？

  • 底层实现：基于 HashMap​，所有元素作为 HashMap​ 的键存储，值统一为 PRESENT​（静态空对象）。

  • 去重机制：

    • 元素的 hashCode()​ 和 equals()​ 方法决定是否重复。
    • 若两个对象的 hashCode()​ 相同且 equals()​ 返回 true，视为重复。

25. ArrayList 和 LinkedList 的区别是什么？

特性 ArrayList LinkedList
底层结构 动态数组 双向链表
随机访问 O(1)（通过索引） O(n)（需遍历链表）
插入/删除 O(n)（需移动元素） O(1)（仅修改指针）
内存占用 较少（仅存储数据） 较高（存储前后节点指针）
适用场景 频繁查询、少增删 频繁增删、少查询

应用示例：

  • ​ArrayList​：分页查询结果列表。
  • ​LinkedList​：实现队列或栈。

27. ArrayList 和 Vector 的区别？

特性 ArrayList Vector
线程安全 非线程安全 线程安全（方法用 synchronized​ 修饰）
扩容机制 容量 ×1.5 容量 ×2
性能 高 低（同步开销）
迭代器 快速失败（Fail-Fast） 快速失败

建议：多线程环境下使用 CopyOnWriteArrayList​ 替代 Vector​。

28. Array 和 ArrayList 有何区别？

特性 Array ArrayList
容量 固定长度 动态扩容
类型支持 基本类型和对象 仅对象（自动装箱/拆箱）
功能方法 无内置方法 提供 add()​, remove()​ 等方法
性能 高（内存连续） 略低（需动态调整）

  • poll() ：队列为空时返回 null​。
  • remove() ：队列为空时抛出 NoSuchElementException​。

30. 哪些集合类是线程安全的？

  • 遗留类：Vector​、Hashtable​、Stack​（不推荐，性能差）。

  • 同步包装类：Collections.synchronizedList()​、synchronizedMap()​。

  • 并发集合：

    • ​ConcurrentHashMap​（分段锁）
    • ​CopyOnWriteArrayList​（写时复制）
    • ​BlockingQueue​（阻塞队列，如 ArrayBlockingQueue​）。

31. 迭代器 Iterator 是什么？

  • 定义：提供遍历集合的统一接口，包含 hasNext()​、next()​、remove()​ 方法。
  • 特点：单向遍历、快速失败（遍历时修改集合会抛出异常）。

示例：

32. Iterator 怎么使用？有什么特点？

  • 使用步骤：

    1. 通过集合的 iterator()​ 获取迭代器。
    2. 使用 hasNext()​ 检查是否还有元素。
    3. 使用 next()​ 获取下一个元素。
    4. 可选 remove()​ 删除当前元素。

  • 特点：

    • 单向性：只能向前遍历。
    • 快速失败：遍历时检测到集合被修改（非通过迭代器自身方法），立即抛出 ConcurrentModificationException​。

33. Iterator 和 ListIterator 有什么区别？

特性 Iterator ListIterator
遍历方向 单向 双向（支持 previous()​）
修改操作 仅 remove()​ 支持 add()​、set()​
索引访问 无 提供 nextIndex()​、previousIndex()​
适用范围 所有集合 仅 List​ 集合
